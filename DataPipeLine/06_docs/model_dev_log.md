# 📔 모델 개발 및 개선 로그 (Model Development Log)

이 문서는 YOLO11m 모델을 활용한 영양제 탐지 프로젝트의 개발 과정, 발생했던 문제점, 그리고 해결 과정을 기록한 개발 일지입니다.

---

## 1. 프로젝트 목적
사용자가 촬영한 사진에서 영양제(Supplement)를 높은 정확도로 탐지하고, 모니터나 키보드 같은 주변 환경에서의 오탐(False Positive)을 최소화하는 강건한 모델 구축.

---

## 2. 개발 단계별 기록

### Phase 1: 프로토타입 구축 (Version 1)
*   **작업 내용**: YOLO11m 기본 모델을 활용하여 일반적인 객체 탐지 성능 확인.
*   **데이터 특징**:
    *   **클래스**: `bottle`, `box` (COCO 데이터셋 기준)
    *   **데이터 규모**: 사전 학습된(Pre-trained) 가중치 활용 및 소량의 샘플 테스트.
*   **개발 의도**: "영양제가 주로 병(Bottle)과 상자(Box)의 형태를 가지므로, 이 둘을 명확히 구분하여 학습시키는 것이 정확도를 높이는 지름길일 것이다."
*   **고민과 문제점**: 
    *   '영양제'라는 특정 도메인에 최적화되지 않아 일반적인 병(Bottle)은 잡지만, 다양한 형태의 영양제 통 식별력이 낮음.
    *   사용자 입장에서는 병이든 상자든 '영양제'로 인식되길 원하는데, 모델이 `box`와 `bottle` 사이에서 클래스를 혼동하며 결과가 불안정해짐.

### Phase 2: 도메인 특화 학습 (Version 2)
*   **작업 내용**: '영양제' 전용 데이터셋을 활용한 본격적인 전이 학습(Transfer Learning).
*   **데이터 특징**:
    *   **데이터 소스**: Roboflow 공개 데이터셋 (`Medicine Detection.v1i`)
    *   **데이터 규모**: 약 300~400장 규모의 정제된 이미지.
    *   **특징**: 하얀 배경이나 스튜디오 조명 아래에서 찍힌 고품질의 영양제 단일 객체 위주.
*   **개발 의도**: "어차피 사용자의 최종 목적은 '영양제'를 찾는 것이다. 형태별로 클래스를 나누는 것보다, 병과 상자 모두를 `supplement`라는 하나의 클래스로 통합하여 학습시키는 것이 모델의 범용성과 사용자 경험 측면에서 유리할 것이다."
*   **고민과 문제점**: 
    *   **배경 오탐**: 깨끗한 데이터로만 학습하다 보니, 사무실이나 거실 같은 복잡한 배경(실생활 노이즈)에서 오탐율이 급격히 상승함.
    *   **클래스 단일화**: 전용 모델 구동을 위해 기존 `bottle`, `box` 체계를 `supplement` 단일 클래스로 통합 완료.

### Phase 3: 데이터 보강 및 오탐 억제 (Version 3)
*   **작업 내용**: 대규모 데이터 병합 및 부정 샘플(Negative Samples)을 통한 고도화.
*   **데이터 특징**:
    *   **총 데이터 규모**: **약 1,450장** (Train 1,160 / Valid 217 / Test 73)
    *   **구성 상세**:
        1.  **기초 데이터**: Version 2의 Roboflow 데이터셋 (약 300장)
        2.  **크롤링 데이터**: `huseong` 폴더 기반 영양제 이미지 (약 880장 승인) - 다양한 브랜드 및 라벨 포함.
        3.  **부정 샘플(Hard Negatives)**: COCO 데이터셋 중 'bottle'이 없는 이미지 (200장) + 랜덤 배경 이미지 (70장)
*   **개발 의도**: "Version 2의 정확도가 떨어지는 근본적인 원인을 파악하자. 깨끗한 배경의 긍정 데이터만으로는 부족하다. 모델이 무엇이 영양제가 '아닌지'를 알 수 있도록 부정 데이터를 보강하고, 다양한 실생활 배경을 강제로 주입하여 일반화 성능을 끌어올려야 한다."
*   **결과 및 분석**: 
    *   **데이터 개선**: 부정 데이터(Negative Samples) 대폭 추가 및 배경 다양성 확보로 오탐율 개선.
    *   **새로운 발견**: 배경 오탐은 현저히 줄었으나, 특정 고해상도 환경(최신 스마트폰 촬영 등)에서만 탐지 성능이 급감하는 기현상 발견.

## 3. 핵심 이슈 분석: 해상도 불일치 (Resolution Mismatch)

프로젝트 도중 발견된 가장 유의미한 문제점은 **학습 데이터와 추론(Inference) 데이터 간의 해상도 격차**였습니다.

### 🔍 문제 현상
*   학습 이미지: 주로 640x640 이하의 저해상도 이미지.
*   추론 이미지: 최신 스마트폰으로 촬영한 **3042 x 4032** 고해상도 이미지.

### 🧬 기술적 분석: Transformer 모델의 해상도 의존성
현대적인 객체 탐지 모델(YOLO11 등)은 내부적으로 **Attention 메커니즘**과 **Transformer inspired blocks (C3k2 등)**을 사용합니다. 이러한 구조에서 발생하는 문제는 다음과 같습니다.

1.  **Scale Invariance의 한계**: 모델은 학습 시 특정 해상도(640px)에서 객체의 특징(Feature)을 배웁니다. 해상도가 10배 가까이 차이 나는 원본 이미지를 640px로 리사이징(Resizing)하는 과정에서, 고주파 정보(High-frequency details)가 손실되거나 객체의 상대적 밀도가 변화합니다.
2.  **Input Resolution Sensitivity**: Transformer 기반 구조는 특징 맵(Feature Map)의 해상도와 객체의 크기 비율에 민감합니다. 학습 시 보지 못한 비정상적으로 높은 해상도의 원본 데이터가 들어오면, 패치(Patch) 단위로 정보를 처리하는 어텐션 필터가 객체를 배경으로 오인하거나 위치를 정확히 특정하지 못하는 현상이 발생합니다. 이를 **용어상 'Input Resolution Mismatch'** 또는 **'Fixed Positional Bias'** 관련 이슈라고 부릅니다.

## 4. 해결 방안 및 향후 계획

### 현재 적용된 해결책
*   추론 시 **이미지 리사이징 전략** 수정 (고해상도 원본의 특징을 유지하며 640px로 최적화).
*   **Hard Negative Mining**: 오탐이 발생한 실제 사무실 환경 사진을 부정 샘플로 추가하여 모델의 식별력 강화.

### 향후 계획 (Version 4)
*   실제 스마트폰 해상도와 유사한 환경의 데이터를 직접 촬영하여 상향된 해상도에서 학습 진행 고려.
*   **Domain-Specific Augmentation**: 손으로 잡고 있는 영양제, 조명 반사가 심한 영양제 등 실전 케이스 보강.

# 📊 YOLO11 영양제 검출 모델 1차 결과 보고서

## 1. 프로젝트 개요
사용자가 업로드한 이미지에서 영양제(병, 박스)를 검출하고, 이를 바탕으로 약물 정보를 제공하기 위한 AI 모델 개발 프로젝트.

## 2. 수행 작업
1.  **데이터 수집**: Selenium 기반 크롤러를 통해 약 400여 장의 영양제 이미지 수집.
2.  **자동 라벨링**: YOLO-World(Zero-shot) 모델을 활용하여 초기 라벨링 자동화.
3.  **모델 학습**: V100 GPU 서버를 사용하여 YOLO11m 모델 100 Epoch 학습 진행.
4.  **서버 연동**: FastAPI 기반의 AI 추론 API 서버 구축 및 학습 가중치(`best.pt`) 적용.

## 3. 학습 수치 (Evaluation Results)
*   **전체 mAP50**: 0.757
*   **Bottle mAP50**: 0.803
*   **Box mAP50**: 0.711
*   *수치상으로는 객체 탐지 모델로서 준수한 성능을 보임.*

## 4. 🧨 실전 테스트 결과 (Issues)
**결과: 불만족 (Unsatisfactory)**

### 주요 문제점:
1.  **배경 오인식 (False Positives)**: 실전 환경(책상 위 등)에서 모니터 암, 안경집과 같은 검은색 사각형 물체를 'Box'로 오인식하는 현상 발생. 
2.  **인식 불가 (False Negatives)**: 특정 각도나 복잡한 배경에서 'Bottle'을 정확히 인식하지 못하는 경우가 잦음.
3.  **환경 불일치 (Domain Gap)**: 크롤링한 깨끗한 제품 이미지(Train)와 실제 복잡한 환경의 사진(Test) 사이의 간극이 커서 적응력이 떨어짐.

## 5. 💡 향후 개선 전략 (Next Steps)

### 전략 A: 2-Stage 파이프라인 도입 (강력 추천)
*   **Stage 1 (Detection)**: Roboflow 등 고품질 범용 데이터셋을 추가 활용하여 '용기(Bottle/Box)' 자체의 검출력을 극대화.
*   **Stage 2 (Verification via OCR)**: 검출된 박스 내에서 텍스트를 추출(Hugging Face 또는 PaddleOCR 활용). 텍스트에서 '영양제 관련 키워드'를 분석하여 2차 필터링 진행.

### 전략 B: Active Learning 및 데이터 보강
*   실제 서비스 환경(복잡한 책상 등)에서 촬영된 데이터를 추가 수집하여 파인튜닝(Fine-tuning) 진행.
*   배경에 아무것도 없는 'Negative Samples'를 추가하여 오탐지 감소 유도.

## 6. 결론
현재의 단일 YOLO 모델만으로는 실전 환경의 변수를 모두 감당하기 어려움. OCR을 결합한 하이브리드 방식이나 실전 환경 데이터의 대폭 보강이 필수적으로 요구됨.
